This guide will walk you through implementing a basic identity verification flow using the MetaMap Android SDK.

## Prerequisites

<Note>
Before starting, ensure you have:
- Completed the [installation](/installation) steps
- Your Client ID from the MetaMap dashboard
- Your Flow ID from the MetaMap dashboard
</Note>

## Implementation Options

The SDK provides two main ways to start a verification flow:

1. **Programmatic Method**: Call `MetamapSdk.startFlow()` from your code
2. **UI Button**: Use the pre-built `MetamapButton` component

Let's implement both approaches.

## Option 1: Programmatic Method

This approach gives you full control over when and how the verification flow starts.

### Step 1: Create Your Activity

Create a new Activity or use an existing one:

```java MethodCallActivity.java
package com.yourcompany.yourapp;

import android.os.Bundle;
import android.widget.Button;
import androidx.appcompat.app.AppCompatActivity;

import com.metamap.metamap_sdk.Metadata;
import com.metamap.metamap_sdk.MetamapSdk;

public class MethodCallActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        Button verifyButton = findViewById(R.id.verifyButton);
        verifyButton.setOnClickListener(v -> {
            MetamapSdk.INSTANCE.startFlow(
                    this,
                    "YOUR_CLIENT_ID",
                    "YOUR_FLOW_ID",
                    new Metadata.Builder()
                            .additionalData("userId", "user_12345")
                            .additionalData("sessionId", "session_xyz")
                            .build()
            );
        });
    }
}
```

### Step 2: Create Layout

Create a simple layout with a button:


### Step 3: Handle Results

Add result handling to your Activity using the modern Activity Result API:

```java MethodCallActivity.java
package com.yourcompany.yourapp;

import android.app.Activity;
import android.content.Intent;
import android.os.Bundle;
import android.widget.Button;
import android.widget.Toast;

import androidx.activity.result.ActivityResultLauncher;
import androidx.activity.result.contract.ActivityResultContracts;
import androidx.appcompat.app.AppCompatActivity;

import com.metamap.metamap_sdk.Metadata;
import com.metamap.metamap_sdk.MetamapSdk;

public class MethodCallActivity extends AppCompatActivity {

    private ActivityResultLauncher<Intent> verificationLauncher;

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        // Register the result handler
        verificationLauncher = registerForActivityResult(
                new ActivityResultContracts.StartActivityForResult(),
                result -> {
                    Intent data = result.getData();
                    if (data == null) {
                        Toast.makeText(this, "Verification cancelled", Toast.LENGTH_SHORT).show();
                        return;
                    }
                    
                    if (result.getResultCode() == Activity.RESULT_OK) {
                        String verificationId = data.getStringExtra(MetamapSdk.ARG_VERIFICATION_ID);
                        String identityId = data.getStringExtra(MetamapSdk.ARG_IDENTITY_ID);
                        
                        Toast.makeText(
                                this,
                                "Verification success! ID: " + verificationId,
                                Toast.LENGTH_LONG
                        ).show();
                        
                        // Send verificationId to your backend
                        sendToBackend(verificationId, identityId);
                    } else {
                        String verificationId = data.getStringExtra(MetamapSdk.ARG_VERIFICATION_ID);
                        Toast.makeText(
                                this,
                                "Verification cancelled. ID: " + verificationId,
                                Toast.LENGTH_SHORT
                        ).show();
                    }
                });

        Button verifyButton = findViewById(R.id.verifyButton);
        verifyButton.setOnClickListener(v -> startVerification());
    }

    private void startVerification() {
        MetamapSdk.INSTANCE.startFlow(
                this,
                "YOUR_CLIENT_ID",
                "YOUR_FLOW_ID",
                new Metadata.Builder()
                        .additionalData("userId", "user_12345")
                        .build()
        );
    }
    
    private void sendToBackend(String verificationId, String identityId) {
        // Send verification result to your backend for validation
        // Your backend should verify the result with MetaMap's API
        // Example: api.verifyUser(verificationId, identityId);
    }
}
```

<Warning>
**Security Note**: Always verify the verification result on your backend server by calling MetaMap's API. Never trust client-side results alone for security-critical operations.
</Warning>

### Step 3: Handle Results (Legacy onActivityResult)

If you're using the legacy `onActivityResult` approach:

```java MetamapButtonActivity.java
package com.yourcompany.yourapp;

import android.app.Activity;
import android.content.Intent;
import android.os.Bundle;
import android.widget.Toast;
import androidx.appcompat.app.AppCompatActivity;

import com.metamap.metamap_sdk.MetamapButton;
import com.metamap.metamap_sdk.MetamapSdk;

public class MetamapButtonActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        MetamapButton button = findViewById(R.id.metamapButton);
        button.setParams(this, "YOUR_CLIENT_ID", "YOUR_FLOW_ID");
    }

    @Override
    protected void onActivityResult(int requestCode, int resultCode, Intent data) {
        if (requestCode == MetamapSdk.DEFAULT_REQUEST_CODE) {
            if (resultCode == Activity.RESULT_OK && data != null) {
                String verificationId = data.getStringExtra(MetamapSdk.ARG_VERIFICATION_ID);
                String identityId = data.getStringExtra(MetamapSdk.ARG_IDENTITY_ID);
                
                Toast.makeText(
                        this,
                        "Verification success! " +
                                "VerificationId: " + verificationId +
                                " IdentityId: " + identityId,
                        Toast.LENGTH_LONG
                ).show();
            } else {
                if (data != null) {
                    String verificationId = data.getStringExtra(MetamapSdk.ARG_VERIFICATION_ID);
                    Toast.makeText(
                            this,
                            "Verification cancelled! ID: " + verificationId,
                            Toast.LENGTH_SHORT
                    ).show();
                } else {
                    Toast.makeText(
                            this,
                            "Verification cancelled!",
                            Toast.LENGTH_SHORT
                    ).show();
                }
            }
        } else {
            super.onActivityResult(requestCode, resultCode, data);
        }
    }
}
```

## Adding Metadata

You can pass custom metadata through the verification flow:

```java
Metadata metadata = new Metadata.Builder()
        .additionalData("userId", "user_12345")
        .additionalData("accountType", "premium")
        .additionalData("sessionId", "session_xyz")
        .build();

// With programmatic method
MetamapSdk.INSTANCE.startFlow(this, "YOUR_CLIENT_ID", "YOUR_FLOW_ID", metadata);

// With MetamapButton
button.setParams(this, "YOUR_CLIENT_ID", "YOUR_FLOW_ID", metadata);
```

<Info>
Metadata is useful for:
- Associating verifications with user accounts
- Tracking verification sessions
- Passing context to your backend
- Segmenting verification analytics
</Info>

## Understanding the Response

When the verification completes, you'll receive:

- **verificationId**: Unique identifier for this verification attempt
- **identityId**: Unique identifier for the user's identity

```java
String verificationId = data.getStringExtra(MetamapSdk.ARG_VERIFICATION_ID);
String identityId = data.getStringExtra(MetamapSdk.ARG_IDENTITY_ID);
```

<Warning>
The result code indicates user action, not verification status:
- `RESULT_OK`: User completed the flow
- `RESULT_CANCELED`: User cancelled the flow

To check actual verification status, query MetaMap's API using the verificationId on your backend.
</Warning>

## Backend Verification

After receiving the verification ID, verify it on your backend:

```java
private void verifyOnBackend(String verificationId, String identityId) {
```

## Common Issues

### Camera Permission Denied

The SDK handles camera permissions automatically. If users deny permission, the flow will be cancelled.

### Invalid Credentials

If you see an error about invalid credentials:
1. Verify your Client ID and Flow ID are correct
2. Check that your MetaMap account is active
3. Ensure the Flow ID corresponds to an active flow in your dashboard

### Integration Not Working

If the verification flow doesn't start:
1. Check that you've completed all [installation](/installation) steps
2. Verify dependencies are synced correctly
3. Check Logcat for error messages
4. Ensure you're testing on API 21+ device

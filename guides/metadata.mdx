The MetaMap SDK allows you to pass custom metadata with verification requests. This is useful for tracking user information, linking verifications to your internal systems, or passing additional context.

## Overview

Metadata is passed using the `Metadata.Builder` class, which supports adding key-value pairs of additional data. This data is associated with the verification session and can be retrieved later through the MetaMap API.

## Using Metadata with startFlow()

When using the programmatic `startFlow()` method, pass metadata as the fourth parameter:

<CodeGroup>

```kotlin Kotlin
import com.metamap.metamap_sdk.Metadata
import com.metamap.metamap_sdk.MetamapSdk

MetamapSdk.startFlow(
    this,
    "YOUR_CLIENT_ID",
    "YOUR_FLOW_ID",
    Metadata.Builder()
        .additionalData("userId", "qwfguweo")
        .additionalData("type", 2)
        .build()
)
```

```java Java
import com.metamap.metamap_sdk.Metadata;
import com.metamap.metamap_sdk.MetamapSdk;

MetamapSdk.INSTANCE.startFlow(
    this,
    "YOUR_CLIENT_ID",
    "YOUR_FLOW_ID",
    new Metadata.Builder()
        .additionalData("userId", "qwfguweo")
        .additionalData("type", 2)
        .build()
);
```

</CodeGroup>

## With Activity Result API

You can combine metadata with the Activity Result API for modern result handling:

<CodeGroup>

```kotlin Kotlin
import androidx.activity.result.ActivityResultLauncher
import androidx.activity.result.contract.ActivityResultContracts
import com.metamap.metamap_sdk.Metadata
import com.metamap.metamap_sdk.MetamapButton

private val resultLauncher: ActivityResultLauncher<Intent> = registerForActivityResult(
    ActivityResultContracts.StartActivityForResult()
) { result ->
    // Handle result
}

val btn = findViewById<MetamapButton>(R.id.metamapButton)

btn.setParams(
    resultLauncher,
    this,
    "YOUR_CLIENT_ID",
    "YOUR_FLOW_ID",
    Metadata.Builder()
        .additionalData("userId", "qwfguweo")
        .additionalData("accountType", "premium")
        .build()
)
```

## Metadata Builder

### Building Metadata

The `Metadata.Builder` class follows the builder pattern:

```kotlin
val metadata = Metadata.Builder()
    .additionalData("key1", "value1")
    .additionalData("key2", 123)
    .additionalData("key3", true)
    .build()
```

```java
Metadata metadata = new Metadata.Builder()
    .additionalData("key1", "value1")
    .additionalData("key2", 123)
    .additionalData("key3", true)
    .build();
```

### Supported Data Types

The `additionalData()` method accepts various data types:

| Type | Example |
|------|----------|
| String | `"username"` |
| Integer | `123` |
| Long | `123456789L` |
| Boolean | `true` or `false` |
| Double | `99.99` |
| Float | `99.99f` |

## Common Use Cases

### User Tracking

Link verifications to your user database:

```kotlin
Metadata.Builder()
    .additionalData("userId", user.id)
    .additionalData("email", user.email)
    .additionalData("accountCreated", user.createdAt)
    .build()
```

### Application Context

Pass application-specific information:

```kotlin
Metadata.Builder()
    .additionalData("accountType", "premium")
    .additionalData("verificationPurpose", "account_upgrade")
    .additionalData("referralCode", "REF123")
    .build()
```

### Session Information

Track session or device details:

```kotlin
Metadata.Builder()
    .additionalData("sessionId", sessionManager.currentSessionId)
    .additionalData("deviceId", deviceInfo.id)
    .additionalData("appVersion", BuildConfig.VERSION_NAME)
    .build()
```

### Transaction Context

Associate verification with a transaction:

```kotlin
Metadata.Builder()
    .additionalData("transactionId", "TXN-12345")
    .additionalData("amount", 500.00)
    .additionalData("currency", "USD")
    .build()
```

## Retrieving Metadata

The metadata you send is stored with the verification session and can be retrieved through:

- MetaMap Dashboard - View metadata in the verification details
- MetaMap API - Access metadata programmatically via the Verifications API
- Webhooks - Receive metadata in webhook payloads

<Note>
Metadata is useful for debugging and tracking verifications in your system. Store identifiers that help you link MetaMap verifications to your internal records.
</Note>
